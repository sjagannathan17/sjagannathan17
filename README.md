## How I think about product

I'm drawn to problems where the user's *first question* is hard to answer well — and where AI changes what's possible. My favorite product moves so far have been:

- **Picking the right verb**, not the right number — replacing a 1–10 risk score with **ER / TODAY / SOON / MONITOR** in a pet-triage app, because owners in crisis need a verb, not a slider. *(Fuzzy-Friend)*
- **Treating cost as a UX feature** — switching to `gpt-4o-mini` made each lit-review run cost $0.01, which let me remove rate limits and unblock iteration. *(LitLens)*
- **Inverting pull-to-push** — shipping email/Slack alerts so a competitive-intelligence team gets the 3 things that matter pushed to them, instead of having to log in and look. *(Flex Practicum)*
- **Saying no to features** to keep a product crisp — refusing to add a "diagnose my pet" mode in Fuzzy-Friend; refusing to add a paper-recommender in LitLens.
